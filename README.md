# レシピ帳

料理中に材料と手順を見失わないための、個人用・静的PWAです。レシピ本文は外部へ送信せず、ブラウザの IndexedDB に保存します。

## ローカルで起動

ブラウザの Service Worker は `file://` では動かないため、フォルダを HTTP サーバーで配信します。Node.js がある場合は、プロジェクトフォルダで次を実行してください。

```powershell
npx serve .
```

表示された URL を開きます。`npx` を使いたくない場合も、任意の静的ファイルサーバーでこのフォルダを配信すれば動きます。

## GitHub Pages で公開

1. このフォルダの中身を新しい GitHub リポジトリへ置く。
2. GitHub の **Settings → Pages** を開く。
3. **Deploy from a branch** を選び、公開するブランチの `/ (root)` を指定して保存する。
4. 表示された Pages URL を iPhone で開き、共有メニューの「ホーム画面に追加」を選ぶ。

ビルド工程は不要です。公開や push、commit はこのプロジェクトから自動では行いません。

## 保存場所とバックアップ

- レシピ本体は、このブラウザ・このサイトURLごとの IndexedDB に保存されます。
- ブラウザのサイトデータ削除、別ブラウザ、別端末には引き継がれません。
- ホーム画面の **JSONバックアップ** で全データを1ファイルへ書き出せます。
- **JSON復元** は有効なバックアップだけを受け付け、既存データを全置換します。復元前に必ず確認します。

## 取り込み解析の規則

- UTF-8 を基本に読み、文字化けらしさが強い場合だけ Shift_JIS の復号結果を採用します。
- 最初の意味のある行、`⌘レシピ`、Markdown見出しを料理名候補にします。
- `材料`、`材料・調味料`、`調味料`、`分量`を材料見出し、`作り方`、`手順`、`STEP`、番号付き工程を手順開始として扱います。
- `人分`、`人前`、`食分`、`分量`表記を何人分候補として拾います。複数あるときも失わず `/` 区切りで表示します。
- `旧レシピ`、`旧版`、`過去レシピ`の語を含む行より後は現在版の解析対象から外します。ただしファイル全文は必ず元テキストに保存します。
- 解析は候補作りです。取り込みプレビューですべて修正してから保存します。

## 既知の制限

- 写真、同期、ログイン、カテゴリ・タグ、買い物リストはありません。
- 解析はローカル規則ベースなので、自由度の高いメモは材料／手順の振り分けを誤ることがあります。元テキストは残り、保存前に修正できます。
- PWAのオフライン起動は、最初に一度オンラインでページを開いてキャッシュを作成したあとに使えます。
- 全体復元は初版では「現在のデータへ追加」ではなく全置換のみです。
